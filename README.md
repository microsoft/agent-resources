# Agent Resources

Curated resources for building, deploying, and managing AI agents with Microsoft tools and platforms — maintained by the [Microsoft Copilot Acceleration Team (CAT)](https://aka.ms/wearecat).

**Front door:** [https://aka.ms/agentresources](https://aka.ms/agentresources)

## What is this?

A collection of five static sites, published together via GitHub Pages from `main` with no build step. Each site is a searchable, deep-linkable accordion hub of high-signal resources — official Microsoft docs and blog posts, Tech Community deep-dives, conference sessions, MVP walkthroughs, and community content — organized for a specific audience.

## Features

Each child site supports:

- **Accordion sections** — collapsible cards grouping resources by topic (Get Started, Adoption, Building, Governance, News, Community, etc.).
- **Sticky navigation strip** — fragment links scroll to and auto-expand the target card.
- **Real-time search** — filters link entries across every section and auto-expands cards with matches.
- **Deep linking** — any section is linkable via `#fragment`; the page opens the card on load.
- **Keyboard + screen reader accessible** — `aria-expanded` state, semantic headings, reduced-motion support.

## Running locally

No build step, no package manager. Any static server works:

```bash
# from the repo root
python3 -m http.server 8000
# then open http://localhost:8000/
```

## Contributing

### Always update `CHANGELOG.md`

Every change to a site's content or design must be reflected in `CHANGELOG.md` with a dated entry grouped by site directory, e.g.:

```markdown
## 2026-04-24

### m365-copilot/
- **News** — added 3 entries: ...

### copilot-studio/
- ...
```

### Keep `main.js` in sync across child sites

All five child sites share **identical** `assets/js/main.js` logic (accordion, search, deep-link). When you change one, copy the file to the other four directories.

### Each site has its own Microsoft Clarity tag

Don't reuse Clarity tag IDs across sites — each has a unique one in the inline `<script>` in `<head>`.

### Images and OG previews

The root site's Open Graph preview lives at `images/og-image-v2.png` (1200×630) and is regenerated by `scripts/generate_og_image.py`. When the product lineup changes, rerun:

```bash
python3 scripts/generate_og_image.py
```

If you refresh the image, bust social-media caches by renaming the file (e.g. `-v3`) and updating the `og:image` / `twitter:image` meta tags in `index.html`, then re-inspect via the [LinkedIn Post Inspector](https://www.linkedin.com/post-inspector/) and [Facebook Sharing Debugger](https://developers.facebook.com/tools/debug/).

├── m365-copilot/            ┐
